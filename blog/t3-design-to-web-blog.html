Web Design

Responsive Web Design

Web development should respond to user's behaviour based on screen size, device and orientation. Simply put, a website should look good, and be easy to use, regardless of the device. Achieving this makes use of Responsive Frameworks that use tools like:

  • CSS Media Queries
  • Responsive Typography that use ems&rems
  • Responsive grid-view that has 12 columns with a total width of 100%, and will shrink and expand as you resize the browser window.

Mobile First Design

Progressive Development wins over Graceful Degradation

With a mobile first standpoint we start with loading the absolute essentials on the smaller platforms. Take the analogy of moving from a small house into a big house. It is far easier and more enjoyable than the reverse move. By starting with a mobile first project that is lean and impressively trimmed down to its most vital elements, the responsive build process to larger devices is not a matter of selecting what to water down, rather what to make more robust!

Frameworks

Framworks such as Bootstraps and Skeleton are each a package made up of a structure of files and folders of standardised code such as HTML, CSS & JS documents which can be used to support the development of websites. The aim of frameworks is to provide a common structure so that developers don’t have to redo it from scratch as, there is no need to reinvent the wheel and can therefore reuse the code provided.

Pros

  • Responsive frameworks
  • Speeds up the mock-up process
  • Clean and tidy code
  • Solutions to common CSS problems
  • Browser compatibility

Cons

  • Mixes content and presentation
  • Unused code leftover
  • You don’t learn to do it yourself

Wireframes

Done very early in the design process, wireframes are basic visual representation of elements on a website. You can think of them as the blueprint to outline and act as a source of reference.

They are important because they establish a hierarcy of information. Following the F shaped rule in which readers eye's travel across a page, starting in the top right corner, the wireframe helps to decide what are the most important aspects of the site.

This is the communication between you and the client. This helps strip down the design idea to its functionality and not get distracted by other elements such as colour and font.

Implementing the wireframe into semantic HTML was difficult but gave a great form of reference and clarified the decisions to be made.
I havn't stayed true to the wireframe in that my content is all left aligned and not centered.
I think left aligned is good for now as it follows the rule of "F" nicely and keeps the site feeling clean.
